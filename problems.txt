# Design Issues

## Dead entities shouldn't disappear
They're omitted from the GMBriefing, but really they ought to be mentioned.
Otherwise players can't loot corpses, etc.

## STALE ROOM DESCRIPTIONS PROBLEM
    The axe_handle_lower description mentions the spider; after it
    flees, the description is factually incorrect.  The briefing
    includes entity state alongside the description, so the LLM can
    mentally override, but this is fragile.  This is a schema-level
    limitation that won't be fixed in Phase 2 but should be kept in
    mind when designing the Context Assembler — it should prominently
    flag entity state next to stale descriptions.

    This is a content/design concern for the Context Assembler (Phase
    5), not a model defect.  The Entity model and BriefingEntity
    already include state fields; the assembler should present them
    prominently alongside descriptions.

## Encounter resolution happens *after* hard-state changes are committed
    Per `phase-4-plan.md` §Step 4, encounters must fire *before* hard-state changes so that a death outcome does not leave the player in a contradictory state (e.g. moved to a new room and died simultaneously).  Current ordering: `apply_hard_changes()` → `resolve_encounter()`.  If the encounter produces `game_over`, the player location has already been updated.  
**Fix:** Reorder to evaluate encounters against the *proposed* changes before committing them; only commit hard changes if the encounter does not produce `game_over` (or commit them conditionally).

## `resolve_examine` completely ignores the `rigorous` flag
    File: `mgmai/engine/resolver.py` (`resolve_examine`)  
    The `rigorous` field on `ExamineAction` is never read.  The schema requires that a rigorous examination evaluate interactions that are gated behind `rigorous` searches. Fix: if `action.rigorous` is `True`, evaluate the target entity/room for any interactions whose availability depends on `rigorous` and include the results in `triggered_narration`.
